API changes

We have recently released a new developer portal called Europeana Labs where all the documentation and information about the Europeana APIs is available. The page you're viewing now is deprecated and will not be updated with any changes. By June 31 2014 this page will be deleted and links to it re-directed to Europeana Labs.

This document describes the changes of Europeana API. The changes are grouped by new API versions. We deploy new versions of the portal and the API quite regularly, but not all new versions has change in the interface. The API documentation always describes the current version of the API. We don't have (yet?) however API call to get the actual version, so the API users should see the date of changes.

Change in version 2.0.6 (2013-08-09)

New allowable value for the profile parameter: "params". When client adds "params" to the profile parameter, the header of the response will contain a params key, which lists the requested and default parameters of the API call. The client can use profile profile parameter in search and object calls. The parameter accepts both single and multiple values separated by comma or space (such as &profile=standard or &profile=standard,params or &profile=standard+params).

Example

http://europeana.eu/api/v2/search.json?wskey=xxxxxxxx&query=mona+lisa&profile=standard+params

returns

{
  "apikey": "xxxxxxxxx",
  "action": "search.json",
  "success": true,
  "requestNumber": 6,
  "params": {
    "query": "mona lisa",
    "profile": "standard params",
    "start": 1,
    "rows": 12
  },
  "itemsCount": 12,
  "totalResults": 195,
  "items": [...]
}
